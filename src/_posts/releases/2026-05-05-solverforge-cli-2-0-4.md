**solverforge-cli 2.0.4** is now available on
[crates.io](https://crates.io/crates/solverforge-cli/2.0.4) with package docs
on [docs.rs](https://docs.rs/solverforge-cli/2.0.4).

This is the current scaffold-target release for generated SolverForge apps.
Fresh projects start on:

- `solverforge 0.11.1`
- `solverforge-ui 0.6.5`
- `solverforge-maps 2.1.4`
- Rust `1.95`

## Verify The Target

```bash
cargo install solverforge-cli --version 2.0.4 --force
solverforge --version
```

Expected output:

```text
solverforge solverforge-cli 2.0.4
CLI version: 2.0.4
Scaffold runtime target: SolverForge crate target 0.11.1
Scaffold UI target: solverforge-ui 0.6.5
Scaffold maps target: solverforge-maps 2.1.4
Runtime source: crates.io: solverforge 0.11.1
UI source: crates.io: solverforge-ui 0.6.5
Maps source: crates.io: solverforge-maps 2.1.4
```

## Generated Dependency Shape

Fresh projects generated by 2.0.4 include:

```toml
[dependencies]
solverforge = { version = "0.11.1", features = ["serde", "console", "verbose-logging"] }
solverforge-ui = { version = "0.6.5" }
solverforge-maps = { version = "2.1.4" }
```

```toml
[runtime]
target = "solverforge 0.11.1"
runtime_source = "crates.io: solverforge 0.11.1"
ui_source = "crates.io: solverforge-ui 0.6.5"
maps_source = "crates.io: solverforge-maps 2.1.4"
```

This supersedes the 2.0.2 scaffold patch for new apps by moving generated
projects from the 0.10 runtime line to `solverforge 0.11.1` while keeping the
current UI and maps package targets explicit.
